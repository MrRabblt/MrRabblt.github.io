面向对象和面向过程的区别？

  • 面向过程：面向过程的性能比面向对象要好。因为类在调用的时候需要实例化，对于性能的开销较大，所以当性能是最重要的考虑因素的时候，比如单片机、嵌入式开发、Linux/Unix等一般采用面向过程开发。但是，面向过程没有面向对象易维护、易复用、易扩展。

  • 面向对象：面向对象易维护、易复用、易扩展。因为面向对象有封装、继承、多态的特性，所以可以设计出低耦合的系统，使系统更加灵活、易于维护。但是，面向对象性能比面向过程低。

    参见 issue : 面向过程 ：面向过程性能比面向对象高？？

但是这并不是根本原因，面向过程也需要分配内存，计算内存的偏移量，Java性能差，的主要原因并不是它是面向对象的语言，而是Java是半编译语言，最终的执行代码并不是可以直接被CPU执行的二进制机械码。

面向对象语言大多都是直接编译成为机械码运行在电脑上，并且其他的一些面向过程的脚本语言不一定性能比Java好。

Java语言有哪些特点？

  1. 易学性
  2. 面向对象(封装、继承、多态)
  3. 可移植性(使用Java虚拟机jvm实现，与平台无关)
  4. 可靠性
  5. 安全性
  6. 支持多线程(C++语言没有多线程机制，因此必须调用操作系统的多线程功能来实现多线程，Java支持多线程)
  7. 网络编程很方便(Java语言诞生本身就是为了简化网络编程设计的，因此Java语言不仅支持网络编程而且很方便)
  8. 编译与解释并存
